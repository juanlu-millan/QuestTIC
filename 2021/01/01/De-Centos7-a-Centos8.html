Actualización de Centos 7 a Centos 8

January 1, 2021

  • En este post veremos como actualizar una máquina con Centos 7 a la versión más reciente de Centos 8 siguiendo una serie de sencillos pasos.
  • Instalamos el paquete epel-release, EPEL son paquetes de Fedora y que no entrán en conflicto con los paquetes de nuestra distribución Linux.
yum install epel-release -y
  • Instalamos en caso de no tenerla instalada las herramientas de yum
yum install yum-utils -y
  • Instalamos RMP que es una herramienta para manejar archivos.
yum install rpmconf -y
  • Ejecutamos el comando de verificación de rpm. 
    rpmconf -a

Verifique los archivos de configuración de todos los paquetes.

  • Hacemos una limpieza de paquetes que no son necesarios.
package-cleanup --leaves
package-cleanup --orphans
  • Ahora instalaremos un gestor de paquetes llamado DNF que sustituirá a yum que ya esta algo desfasado.
yum install dnf
  • Llega el momento en el que desinstalamos yum y eliminamos su directorio.
dnf -y remove yum yum-metadata-parser

rm -Rf /etc/yum
  • Habilitamos los repositorios de Centos 8 ( http://mirror.centos.org/centos/8/BaseOS/x86_64/os/Packages/)
dnf install \
http://mirror.centos.org/centos/8/BaseOS/x86_64/os/Packages/centos-repos-8.2-2.2004.0.1.el8.x86_64.rpm \
http://mirror.centos.org/centos/8/BaseOS/x86_64/os/Packages/centos-release-8.2-2.2004.0.1.el8.x86_64.rpm \
http://mirror.centos.org/centos/8/BaseOS/x86_64/os/Packages/centos-gpg-keys-8.2-2.2004.0.1.el8.noarch.rpm
  • Actualizamos el paquete epel que vimos anteriormente.
dnf upgrade -y epel-release
  • Generamos la caché.
dnf makecache
  • Eliminamos los ficheros temporales realizando una limpieza.
dnf clean all
  • Debemos eliminar las versiones actuales del kernel

Comprobamos el instalado.

 rpm --query centos-release
Con rpm -qa egrep kernel nos muestra tanto el kernel-core como kernel-tools-libs y kernel-tools de la versión actual y la eliminamos. 
rpm -e `rpm -qa | egrep kernel`
  • Eliminamos los paquetes que tienen conflictos.
rpm -e --nodeps sysvinit-tools
  • Actualizamos de nuevo.
dnf -y --releasever=8 --allowerasing --setopt=deltarpm=false distro-sync
  • Instalamos el kernel-core de Centos8
dnf install -y kernel-core
  • Instalamos los paquetes minimos del sistema:
dnf -y groupupdate "Core" "Minimal Install" \
--allowerasing --skip-broken
  • Reiniciamos y realizamos la comprobación:
sudo reboot
  • Comprobación:
[centos@quijote ~]$ uname -r
4.18.0-193.28.1.el8_2.x86_64
[centos@quijote ~]$ cat /etc/redhat-release
CentOS Linux release 8.2.2004 (Core)
